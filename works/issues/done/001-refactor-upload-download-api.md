- [x] 파일 다운로드 API가 `uploads` 폴더에서 파일을 읽어 제공하도록 수정
- [x] JPA 연동 없이, 순수 파일 시스템 기반으로 동작

## 테스트 시나리오

### 파일 업로드 (POST /upload)
- **성공 케이스**:
  - [x] 정상적인 파일을 업로드하면 `200 OK` 응답과 함께 파일명과 파일 크기를 반환한다.
  - [x] 업로드된 파일은 `uploads` 폴더에 UUID가 붙은 고유한 이름으로 저장된다.
- **실패 케이스**:
  - [x] 파일을 첨부하지 않고 요청하면 `400 Bad Request` 응답을 반환한다.

### 파일 다운로드 (GET /download/{fileName})
- **성공 케이스**:
  - [x] 업로드된 파일의 저장된 이름으로 요청하면 `200 OK` 응답과 함께 파일이 다운로드된다.
  - [x] 다운로드 시 `Content-Disposition` 헤더에 올바른 파일명이 포함되어 있는지 확인한다.
- **실패/엣지 케이스**:
  - [x] 존재하지 않는 파일명으로 요청하면 `404 Not Found` 응답을 반환한다.
  - [x] 파일명에 `../` 와 같은 경로 조작 문자가 포함된 경우, 이를 차단하고 `400 Bad Request` 또는 `404 Not Found`를 반환하여 경로 탐색 공격을 방지한다. (보안 강화)
